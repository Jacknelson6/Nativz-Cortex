## How Skills Work

**Skills** are markdown-based instruction sets that Claude Code loads on demand. When you invoke a skill (e.g., `/copywriting`), Claude reads the SKILL.md file and follows its structured guidance for that specific domain.

**Plugins** provide infrastructure — MCP servers, hooks, agents, and subagent systems that extend Claude Code's capabilities at a lower level.

**Vercel skills** are injected automatically when you touch relevant files (e.g., editing a Next.js route handler triggers the Next.js skill).

### Installation Locations

```
~/.claude/skills/              → Global skills (available in all projects)
~/.claude/plugins/             → Plugins (system-level extensions)
.claude/skills/                → Project-specific skills
.claude/commands/              → Project-specific slash commands
```

### Skill Sources

- **Superpowers Plugin** — Development workflow discipline (claude-plugins-official)
- **Vercel Plugin** — 40+ platform skills auto-injected by file pattern
- **Ars Contexta** — Knowledge management system (standalone plugin)
- **Marketing Skills** — Installed individually to `~/.claude/skills/`
- **n8n Skills** — Workflow automation guidance
- **Custom Project Skills** — Per-project patterns in `.claude/skills/`
