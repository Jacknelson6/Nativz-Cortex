# Impeccable — Design Context

This file is the source of truth for design decisions in Cortex. All `/impeccable *` skills read from here before doing any work. Keep it current.

---

## Design Context

### Users

**Two audiences, equal priority. Portal is the more fragile of the two — optimize for it first when they conflict.**

- **Portal (clients, role=`viewer`)** — Nativz' paying customers. They log in occasionally to see research, video ideas, briefs, and what's been scheduled. Light cognitive load. Glancing, not operating. Must feel like a client-facing deliverable, not an internal CMS. They should walk away thinking *"my agency is smarter than I realized."*
- **Admin (internal team, role=`admin`/`super_admin`)** — Nativz strategists, videographers, content leads. Daily operators. Dense. Keyboard-heavy. Running topic search, building topic plans, QA'ing AI output, scheduling posts. Can tolerate — and reward — more information density, more controls, more operator-y UI. The job-to-be-done: walk onto set knowing exactly what to film.
- **Sidebar navigation is a keeper.** Jack likes it and how it works. Don't rebuild sidebar patterns; extend them.

### Brand Personality

**Three words: Nerdy · Intelligent · Confident**

Not "modern," not "clean," not "professional." Cortex is a research tool for people who take short-form video seriously. The tone should read as *"we did the reading"* — citations, evidence, actual numbers, transparent sourcing, unashamedly opinionated output. Confidence without swagger. Nerdy without being cute or ironic.

**Emotional targets:**

- **Team feels:** ready, confident, prepared, knowledgeable. Like they've been briefed by a sharper analyst before every shoot.
- **Clients feel:** impressed, informed, in the loop — peace of mind that an enormous amount of intelligence is backing their agency. The portal should quietly radiate *"these people know what they're doing."*

### Aesthetic Direction

**Theme:** Dark by default for both admin and portal (operator + late-evening client browsing both favor dark). Nativz brand mode is the primary. Anderson Collaborative is a first-class alternate brand mode (`data-brand-mode="anderson"`) that flips to teal/light. Future client agencies will get their own brand mode via the same pattern.

**Anti-references:** None given. Avoid defaults (Linear-clone minimal, Vercel-dark-with-gradients, Notion-with-icons) by execution quality, not by rule.

**Nativz brand system is the source of truth** (see "Tokens" below). The current `app/globals.css` uses older interim values (Geist, `#046bd2`, `#8b5cf6`) — treat those as drift. New work should reach for the canonical Nativz tokens. Don't refactor existing screens just to swap tokens; reconcile as screens are touched.

### Design Principles

1. **Portal feels like a deliverable; admin feels like a cockpit.** Same visual language, different density. A portal page should read like a report handed to a client. An admin page can read like a workbench with sharp tools laid out in reach.

2. **Confidence through evidence, never through polish alone.** Show counts, sources, timestamps, model names, score breakdowns. Cortex is research software — hide the machinery and it looks like another AI slop generator. Expose it tastefully and it looks like we did the reading.

3. **Nerdy details earn their place.** Typography with intent, monospace for actual machine-generated values (IDs, model names, timestamps), cyan highlighter underlines for signature emphasis on hero copy. Don't decorate — instrument.

4. **Purple is CTA. Cyan is brand. Coral is accent. Don't blur them.** `#9314CE` is the "do this now" color. `#00AEEF` is the "this is Nativz" color. `#ED6B63` is rare highlight / form urgency. When everything is emphasized, nothing is.

5. **Motion is the pacing layer, not decoration.** Use the existing motion tokens (`--ease-out-expo`, `--duration-*`, stagger patterns in `app/globals.css`). A well-orchestrated page load with staggered reveal beats scattered micro-interactions. Never animate layout properties; `transform` + `opacity` only. Respect `prefers-reduced-motion` (already wired).

## Nativz Brand Tokens (source of truth)

**When brand tokens in code disagree with this section, this section wins.** Use the `--nz-*` names below; they're the names that will carry forward into refactored brand modes.

### Fonts

- **Display:** Jost (open-source Futura PT substitute). Geometric-humanist sans, closed `a`, same vertical proportions as Futura PT. Primary for H1–H4, hero, buttons, eyebrows.
- **Body:** Poppins (captions, long-form). Light weight for hero body.
- **Sans (UI):** Rubik.
- **Editorial:** Roboto Slab (rare).
- Import: Google Fonts `Jost:300–800`, `Poppins:300–700`, `Rubik:300–700`, `Roboto Slab:400,600,700`.
- **Button text:** Jost 700, uppercase, letter-spacing `2px`.
- **"Bold" in Nativz world = 600, not 700.** 700 is "heavy."

### Signature: cyan highlighter underline

Wrap emphasis words in `<u>` inside H1/H2. The underline is a **thick translucent cyan bar behind the glyphs** (negative offset, not below text). This is a Nativz tell — don't water it down.

```css
.nz-u, :where(.nz-hero, .nz-h1, .nz-h2) u {
  text-decoration: underline;
  text-decoration-color: rgba(0, 130, 181, 0.70);  /* base hex #0082b5 */
  text-decoration-thickness: 35px;   /* 24px on h2 */
  text-decoration-skip-ink: none;
  text-underline-offset: -15px;      /* -8px on h2 */
}
```

### Spacing (4pt-ish, observed rhythm 10/20/40/60/100)

`--nz-space-1..9` → `4 / 8 / 12 / 20 / 30 / 40 / 60 / 100 / 150`. Use tokens, not raw pixels.

### Radius

- **Action buttons — FULLY ROUNDED PILL** (`border-radius: 9999px`). Confirmed against live nativz.io — every CTA on the marketing site is a pill, not a rounded rectangle. This is a Nativz signature; **do not substitute with 5/10/20px corners on buttons**.
- `--nz-radius-sm: 5px` — inline chips, small tags
- `--nz-radius-md: 10px` — cards, media, modals
- `--nz-radius-lg: 20px` — hero panels, large feature surfaces

### Icon tiles — full circle

Colored icon backings (the filled shapes behind an icon, next to labels or in service blocks) are **full circles** (`rounded-full`), not rounded squares. Live-site-confirmed. Cortex dashboards were using `rounded-xl` — correct treatment is circle.

### Eyebrow text — signature Nativz move

Every H1/H2 section on the live site is preceded by a **small cyan italic line** of intro/tagline copy. Example:
> *"We're committed to driving brand success through expert social media advertising and data-driven marketing strategies."*

Render spec: Jost 500–600, ~16–18px, italic, `--nz-cyan` color, sits directly above the section heading with ~8px gap. Utility class `.nz-eyebrow` in `globals.css`. Use sparingly on dashboard sections where a one-line context note belongs.

### Elevation

Nativz is **flat — zero resting shadows, confirmed against live marketing site**. Cards sit directly on the surface with no drop shadow. Shadows appear ONLY on:
- Hover lift (subtle 4–8px y-offset + soft blur)
- Sticky/floating surfaces (toasts, dropdowns, modals)
- Active / pressed state on CTAs

Don't reach for generic drop shadows on cards by default. A flat card on a darker background is the Nativz aesthetic.

## Notes for future impeccable calls

- Anderson Collaborative mode exists and is locked to client brand (teal `#36D1C2`, light bg). Don't touch its tokens — they come from AC's official guide.
- `docs/detail-design-patterns.md` catalogs 56 micro-interaction patterns already in use. Reference it before inventing a new pattern.
- Motion tokens (`--ease-out-expo`, `--duration-fast/normal/slow`) are defined globally; reuse them.
- Sentence case everywhere for copy (per CLAUDE.md). No Title Case UI labels.
- Dark theme default; don't switch to light unless building for a brand mode that demands it.
